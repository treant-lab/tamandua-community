# Governance

Tamandua uses a maintainer-led governance model while the project is early.

## Maintainers

Maintainers are responsible for:

- setting public project boundaries;
- reviewing community process changes;
- triaging cross-repository proposals;
- routing implementation issues to component repositories;
- enforcing the Code of Conduct;
- coordinating security disclosure.

## Decision Types

| Decision | Where it should happen |
| --- | --- |
| Bug in one component | Component repository issue or pull request |
| Cross-component proposal | `tamandua-community` proposal or RFC |
| Detection idea | Discussion or Detection Proposal issue |
| Vulnerability | Private security email |
| Public roadmap change | Discussion, proposal, then roadmap update |
| Release or versioning process | Component repository plus community announcement |

## RFCs

Use `rfcs/` for decisions that affect more than one repository or change long-term project direction.

An RFC should include:

- summary;
- motivation;
- affected repositories;
- proposed behavior;
- alternatives considered;
- rollout plan;
- security and privacy considerations;
- open questions.

## Conflict Resolution

Maintainers may ask contributors to narrow scope, move implementation to a component repo, split a proposal, or defer a change until security review is complete.

Security, privacy, response safety, and supply-chain integrity take precedence over speed.
