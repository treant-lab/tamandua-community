# Discord and GitHub Discussions

Discord is for fast coordination. GitHub remains the durable source of truth.

Permanent invite: https://discord.gg/52Duv5GVDF

## Use Discord For

- onboarding;
- quick setup questions;
- support triage;
- demos;
- contributor coordination;
- benchmark discussion before filing durable results.

## Use GitHub Discussions For

- Q&A that should be searchable;
- roadmap proposals;
- benchmark interpretation;
- detection design;
- community announcements;
- contributor planning.

## Use GitHub Issues For

- reproducible bugs;
- actionable feature requests;
- telemetry gaps with examples;
- false-positive reports;
- benchmark results with artifacts.

## Do Not Use Public Channels For

- private keys;
- credentials;
- tokens;
- customer telemetry;
- live malware samples;
- exploit payloads;
- vulnerability details before coordinated disclosure.

- `#announcements`

Keep the launch channel list small. Add private or partner channels only when there is a clear moderation need.
