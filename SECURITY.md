# Security Policy

Please do not report security vulnerabilities through public GitHub issues, discussions, pull requests, Discord, or other public channels.

Report vulnerabilities by email:

**security@treant.io**

## What to Include

- Vulnerability type.
- Affected component or repository.
- Impact and attack scenario.
- Reproduction steps.
- Relevant source paths, versions, commits, or configuration.
- Proof of concept if safe to share privately.
- Suggested fix, if available.

Do not include sensitive customer data unless explicitly coordinated with maintainers.

## Response Targets

| Action | Target |
| --- | --- |
| Initial acknowledgment | 48 hours |
| Preliminary assessment | 7 days |
| Status update during investigation | Every 14 days |
| Coordinated public disclosure | Usually within 90 days, or after a fix is available |

Critical production-impacting reports are prioritized.

## Safe Harbor

We consider good-faith security research conducted within this policy to be authorized. We will not pursue legal action for accidental, good-faith violations when the researcher avoids privacy harm, avoids persistence, avoids data destruction, and reports promptly.

If you are unsure whether an activity is permitted, email us before proceeding.

## Public Channels

Public channels may be used for:

- general hardening ideas;
- safe configuration questions;
- sanitized benchmark results;
- non-sensitive bug reports.

Public channels must not be used for:

- exploitable vulnerability details before disclosure;
- credentials, tokens, private keys, or dumps;
- live malware samples;
- exploit payloads;
- private customer telemetry.
